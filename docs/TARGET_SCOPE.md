# 目标范围与产品策略（小而精）

## 先回答你的核心问题

可以做全球 + 多类型（高校赛、企业赛、国际赛），但不建议一开始全量铺开。

原因：
- 站点结构差异大，维护成本会指数增长
- 全量做会牺牲准确率与稳定性
- 用户真正高频使用的，通常只是一部分高价值赛事

## 推荐方向：先小而精，再扩张

### MVP（第一阶段）

只做一个高价值切片：
- 地域：以中国为主，补充少量全球头部赛事
- 类型：高校赛 + 企业赛
- 专业：计算机 / AI / 信息安全 / 数学建模
- 等级：国家级、国际级优先

目标：
- 保证抓取稳定
- 保证字段准确
- 保证查询体验清晰

### 第二阶段

扩到 30-50 个来源，增加：
- 商科、电商、机器人、设计等赛道
- 更完整的企业赛事
- 增强检索和筛选

### 第三阶段

扩到 80+ 来源，增加：
- 全球赛事覆盖
- 个性化订阅和提醒
- 日历与截止预警

## 分类体系（统一标准）

建议所有赛事都按这 4 个维度归类：

1. 专业方向（discipline）
- 计算机
- 人工智能
- 信息安全
- 数学建模
- 电子电气
- 机器人
- 商科电商
- 设计创意
- 综合

2. 级别（level）
- 校级
- 省级
- 国家级
- 国际级

3. 主体类型（host_type）
- 高校/学会
- 政府/官方机构
- 企业
- 国际组织

4. 参赛对象（audience）
- 大学生
- 研究生
- 中学生
- 公开组

## 为什么这套适合“打破信息差”

- 用户找比赛时第一需求是“能筛选”，不是“全量堆信息”
- 分类统一后，才可以做靠谱的导航、订阅和提醒
- 先把高价值赛道做深，用户感知最强

## 你现在最该做的选择

如果你想小而精，建议先定这个定位：

“面向中国大学生的高价值科技竞赛雷达，聚焦计算机/AI/安全/建模，优先国家级与国际级，并补充头部企业赛。”

这个定位最容易快速做出口碑。
